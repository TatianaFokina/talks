The clamp() definition

The CSS function for setting up the range of values for some CSS properties.

clamp() anatomy

Available values are:

  • Minimum (minValue)
  • Preferred (prefValue)
  • Maximum (maxValue).
				clamp(minValue, prefValue, maxValue);

A preferred value enables smooth text size adjustments.

Preferred width (vw)

v = 100 ⋅ ( y 2 − y 1 ) x 2 − x 1
  • y1 ‚Äî the min font size
  • y2 ‚Äî the max font size
  • x1 ‚Äî the min breakpoint (here the min font size ends)
  • x2 ‚Äî the max breakpoint (here the max font size begins).

Relative size (rem)

r = ( x 1 ⋅ y 2 ) − ( x 2 ⋅ y 1 ) x 1 − x 2
  • y1 ‚Äî the min font size
  • y2 ‚Äî the max font size
  • x1 ‚Äî the min breakpoint (here the min font size ends)
  • x2 ‚Äî the max breakpoint (here the max font size begins).

Approach with the formula

  • More smoother scaling
  • Text isn‚Äôt too small on mobile devices
  • Text doesn‚Äôt become larger too abruptly as the screen width increases.

Part NaN: Don‚Äôt touch it when it works

Success in fluid approach

  • Don‚Äôt set the min and the max font sizes in pixels
  • Keep an eye on the ratio between the largest and the smallest font sizes.

Why wouldn‚Äôt we use pixels?

If you set the font size in pixels, the browser‚Äôs default text size (16 pixels) will be overridden, and text won‚Äôt scale when user settings change.

Example with the html element

				  font-size: 15px;
				}
				h1 {

Ratio between larger and smaller values

The max font size must be less than or equal to 2.5 times the min font size ü§Ø

Why this ratio?

Text with inappropriate sizing doesn‚Äôt scale well because it won‚Äôt achieve a proper increase due to overlap between zoom and scaling functions.

Who resizes and rescales the UI?

  • People with different eye conditions
  • Those who want to make font and elements larger for any reason.

Part 5. Wrapping up

Fluid typography doesn‚Äôt replace responsive typography. The choice depends on the specific case.

Fluid typography

Pros ‚ûï

  • Doesn‚Äôt require supporting a large set of breakpoints
  • Achieved with a single line of CSS
  • You don‚Äôt need to create separate design layouts for every screen size.

Cons ‚ûñ

  • The calculation logic for the preferred size is difficult
  • Can break browser page zoom functionality
  • Text is unpredictable, so the final layout may differ from the expected results.

Fluid typography use cases

Any heading, especially large ones, and texts with noticeable size differences.
